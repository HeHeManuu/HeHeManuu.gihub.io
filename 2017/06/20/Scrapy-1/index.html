初识Scrapy

发表于 2017-06-20 | 分类于 爬虫 | |

爬虫概念

网络爬虫，是一种按照一定的规则，自动地抓取万维网信息的程序或者脚本。根据不同的任务可分为通用爬虫，主题爬虫等。爬取的策略有广度优先，深度优先(通常很少用)，最佳优先(按照种子链接的主题相关性进行排序)。
最简单的爬虫只需网络访问和页面解析工具即可，比如利用urllib2和BeautifulSoup即可完成简单的抓取任务。

Scrapy

Scrapy，是用Python开发的一个快速,高层次的网页抓取框架，用于抓取web站点并从页面中提取结构化的数据。Scrapy用途广泛，可以应用在包括数据挖掘，信息处理等领域中。Scrapy是基于事件网络驱动Twisted框架编写的，它对于并发采用非阻塞的方式处理。

Scrapy框架

Scrapy主要包含以下组件。

Scrapy Engine

引擎负责控制系统所有组件中的数据流，和特定的动作发生后触发事件。

Scheduler

调度器接受引擎的请求，并将其入队，以便在引擎请求他们时候返回。

Downloader

下载器获取网络页面，将其返回给引擎，然后引擎再传递给spider。

Spiders

Spiders是Scrapy用户定制解析和抽取信息的类。

Item Pipeline

Item Pipeline处理被Spiders提取出来的items，包括清洗，验证和持久化。

Downloader middlewares

下载中间件放置在引擎和下载器中间，处理引擎传给下载器的请求，以及下载器传给引擎的回复。我们可以根据自己项目中的需求对下载中间件进行定制。

Spider middlewares

这放置在引擎和Spiders中间，处理爬虫的输入和输出。

Scrapy Data flow

图片来自于Scrapy官方文档。




数据流由引擎所掌控。按照图上的标号依次解释如下：

  1. 引擎从Spider获得初始的请求。
  2. 引擎调度Scheduler中的请求，并且请求下一个爬取的请求。
  3. Scheduler返回下一个请求给引擎。
  4. 引擎发送请求给Downloader，请求穿过下载中间件。
  5. 一旦Downloader完成网页的下载，将回复传递给引擎，并且穿过下载中间件。
  6. 引擎收到回复后，将此发送给Spider进行处理。传递通过Spider中间件。
  7. Spider处理完回复后产生抽取的items和新的请求，一并传递给引擎。同样穿过Spider中间件。
  8. 引擎将items发送给Item Pipelines,将请求发送给调度器，并请求下一个爬取的请求。
  9. 处理将会一直执行，直到调度器没有请求为止。

定制爬虫

编写Spider

需要根据自己的具体的需求，进行爬虫的编写。

  1. 需要设置爬虫的名称，爬虫初始种子URLs。
  2. 需要配置一些访问规则，比如利用Rule编写相关的URL匹配规则。
  3. 需要编写相关的item提取方法。

编写item

根据自己具体需求将网页结构化，这里需要提前设定相关结构化的属性。

编写pipelines

可以编写不同的pipelines来处理item，以将item保存为不同的形式。比如文本，数据库等。

配置settings

根据不同的爬取任务来进行Scrapy的参数设定。比如下载延迟，网页链接深度限制，任务终止条件，各个定制的中间件优先级设定等。
